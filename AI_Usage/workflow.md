# AI-assisted Development Workflow

The development of this project followed a human-in-the-loop workflow where AI systems assisted different stages of the project while all scientific decisions remained under manual control.

---

# Phase 1 — Literature Review

**Primary Tool:** NotebookLM

Activities:

- Upload mentor-provided documents.
- Read research papers.
- Summarize literature.
- Understand causal inference concepts.
- Organize methodological references.

Output:

- Better understanding of project objectives.
- Literature summaries.
- Methodological notes.

---

# Phase 2 — Project Planning

**Primary Tool:** ChatGPT

Activities:

- Design project architecture.
- Plan synthetic benchmark.
- Plan real-world validation.
- Design repository structure.
- Discuss causal methodology.

Output:

- Project workflow.
- Dataset strategy.
- Repository organization.

---

# Phase 3 — Dataset Construction

**Primary Tool:** ChatGPT

Activities:

- Synthetic SEIR dataset planning.
- Real-world dataset selection.
- Variable mapping.
- Missing-value strategy.
- Weekly panel construction.

Output:

- Benchmark dataset.
- Real observational dataset.

---

# Phase 4 — Causal Inference

Activities:

- Propensity Score Matching
- Inverse Probability Weighting
- Augmented IPW
- DoWhy
- EconML

AI assisted with implementation guidance while all estimator selection, parameter tuning, debugging, and interpretation were manually performed.

---

# Phase 5 — Documentation

Activities:

- LaTeX report drafting.
- README preparation.
- GitHub organization.
- AI documentation.

---

# Human Validation

Every AI-generated suggestion underwent manual verification before being incorporated into the project.

Validation included:

- reviewing statistical assumptions,
- verifying implementation,
- testing generated code,
- interpreting outputs,
- confirming causal reasoning.

No AI-generated output was accepted without manual review.
